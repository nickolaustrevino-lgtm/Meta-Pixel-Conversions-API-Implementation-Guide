Foundation

Overview: Browser + Server Tracking

Why two tracking methods are required and how they complement each other.

🌐
Browser Event
Meta Pixel (Client-Side)

The Meta Pixel is a JavaScript snippet installed on your funnel. When a visitor submits a form or takes an action, the pixel fires an event directly from their browser to Meta. Limitation: Browser pixels are blocked by ad blockers and iOS privacy changes. 20–40% of conversions go unmeasured without server-side tracking.

💻
Server Event
Conversions API (Server-Side)

CAPI sends event data directly from GoHighLevel's servers to Meta, bypassing the browser entirely. It is unaffected by ad blockers or iOS restrictions. Result: Full conversion visibility and more reliable attribution when used alongside the pixel.

Why Both Are Required
Meta recommends dual-tracking (pixel + CAPI) simultaneously. The browser pixel captures real-time user signals. The server event captures reliable conversion data regardless of browser restrictions. Event deduplication (Section 6) ensures Meta counts each conversion exactly once.

Implement Event Deduplication

The most common tracking configuration error — and how to prevent inflated conversion counts.

🚨
The #1 Tracking Mistake
When both the browser pixel AND the CAPI server event fire for the same conversion without a shared event_id, Meta counts that conversion twice. This inflates reported conversions, distorts ROAS, and causes Meta's algorithm to optimize incorrectly. Every browser event and its corresponding server event must share the same event_id.

Data Quality

Improve Event Match Quality (Advanced Matching)

Send user identifiers with each event so Meta can accurately attribute conversions to the correct ad.

Meta's Event Match Quality (EMQ) score measures how reliably it can match conversion events to Meta user accounts. A higher EMQ score means more attributed conversions, better audience modeling, and more effective optimization. Including user identifiers dramatically improves this score.

✅
Why This Matters
GoHighLevel's CAPI action collects contact field values automatically. Mapping form fields to Meta parameters takes under 5 minutes and can move your EMQ score from "Poor" to "Great" — resulting in significantly better attribution and lower CPL over time.

⚠️
Hashing Is Automatic
GoHighLevel automatically hashes all customer information parameters (email, phone, name) using SHA-256 before transmitting to Meta. Do not manually hash these values inside the workflow — doing so causes double-hashing and the data becomes unreadable by Meta.

Advanced Signal Strategy

Sending Qualified Lead Signals

Tell Meta which leads are worth finding more of — improving lead quality and ROAS.

The QualifiedLead event fires only when a lead meets specific qualification criteria. This signal is more valuable than a raw Lead event because it represents a smaller, higher-intent audience Meta can model from — directly improving lead quality over time.

When to Trigger QualifiedLead in GoHighLevel
A
Trigger: Consultation Attended
Workflow trigger: Appointment status changes to Attended / Showed. This confirms engagement beyond just booking.
Automation → Trigger: Appointment Status Changed → Status: Attended
B
Trigger: Pipeline Stage Updated to "Qualified"
Fire QualifiedLead when a contact's CRM pipeline stage is moved to a designated Qualified stage — manually or via automation.
Automation → Trigger: Pipeline Stage Changed → Stage: Qualified
C
Trigger: Consultation Booked (High Intent)
For tight sales cycles, booking alone may qualify as a QualifiedLead. Use the same trigger as Workflow 2 with a separate CAPI action sending QualifiedLead.

Monitoring Diagnostics

Where to check tracking health post-launch and how to resolve the most common issues.

Events Manager Diagnostics is the primary tool for identifying tracking issues after launch. Check it weekly during the first month of campaign operation. Issues left unresolved degrade ROAS over time as Meta's algorithm receives incomplete signal data.

1
Navigate to Diagnostics Tab
In Meta Events Manager, select your dataset, then click the Diagnostics tab. Issues are grouped by severity: Errors (red), Warnings (yellow), Suggestions (blue).
Events Manager → Dataset → Diagnostics Tab
2
Review Event Match Quality Scores
Under the Overview tab, review EMQ scores for each event type. Scores below 6.0 indicate insufficient customer data. Return to Section 7 to add missing parameters.
Events Manager → Dataset → Overview → Event Match Quality
