SECTION 01

Architecture Overview & Platform Roles Updated

What changed from the GoHighLevel-based setup, why Stripe is the correct CAPI source, and how each platform fits into the tracking stack.

⚡
v2.0 Key Change: GoHighLevel Removed as CAPI Trigger

GoHighLevel is the NowFit phone system and a supplementary sales CRM. It is not where payments occur, and it is not where CAPI events should originate. All client payments run through Stripe via the NowFit proprietary platform. Stripe is the sole authoritative CAPI source for Purchase events. This is simpler, more accurate, and does not require GoHighLevel automation workflows.

What the Meta Pixel Does

The Meta Pixel is a JavaScript snippet embedded in the <head> of Monet's HTML site. It fires browser-side events (PageView, Lead, Purchase) directly from the visitor's browser to Meta's servers. Browser events capture top-of-funnel behavior and can be affected by ad blockers or iOS privacy restrictions.

What Conversions API Does

The Meta Conversions API (CAPI) sends events server-to-server, bypassing the browser entirely. In this architecture, the NowFit development team configures a Stripe webhook listener. When Monet's client completes payment in Stripe, the webhook fires and NowFit's server immediately sends a verified Purchase event to Meta via CAPI, including full customer data and revenue value.

Why Both Are Required

Pixel handles top-of-funnel behavior. Stripe CAPI delivers the verified bottom-of-funnel signal with payment-confirmed data. Running both together (hybrid tracking) maximizes signal coverage. Deduplication via a shared event_id ensures Meta counts each conversion exactly once.

Add the following to the form submit handler. This generates a unique eventID at submission time and captures the fbc and fbp values that must be stored alongside the lead in the NowFit CRM so they pass through to Stripe metadata at payment.

Lead Event — form submit handler
// Unique event ID for this lead — required for CAPI deduplication
const eventID = 'lead_' + Date.now() + '_' + Math.random().toString(36).substr(2, 9);
document.getElementById('meta_event_id').value = eventID;

// Capture fbc (from URL) and fbp (from cookie) — pass these to NowFit CRM on lead creation

const fbc = urlParams.get('fbclid')

const fbp = document.cookie.match(/_fbp=([^;]*)/)?.[1] || '';

// Store in hidden form fields or send to NowFit CRM endpoint

document.getElementById('meta_fbp').value = fbp;

SECTION 06

Event Deduplication — Stripe + Pixel Critical

When both the browser pixel and the Stripe CAPI webhook fire for the same purchase, Meta must count it only once. This is controlled entirely by the event_id / eventID field.

🚨
The #1 Tracking Mistake

If the browser and server events don't share a matching event_id, Meta counts every conversion twice. This inflates reported ROAS, distorts optimization, and makes it impossible to trust performance data. Fix before launch — not after spend.

The Stripe Deduplication Pattern

Stripe's checkout.session.id (format: cs_live_xxxx...) is unique per transaction and available on both paths: the browser receives it via the ?session_id= URL parameter on the confirmation page; the server receives it in the webhook payload. Use it as the event_id in both places.

SECTION 07

Advanced Matching: fbc, fbp, and Customer PII Updated

Stripe provides verified, billing-confirmed customer data at the point of payment. This is higher quality than any form-based source. Passing these identifiers improves Meta's event match quality, attribution accuracy, and lookalike audience performance.

✓
Stripe Advantage for Matching

Stripe's customer object contains confirmed email, name, phone, and billing address. These are more reliable than unverified form inputs. Passing them to Meta's CAPI user_data fields directly improves the Event Match Quality score on the Purchase event — the single most important factor in CAPI attribution accuracy.

⚠️
SHA-256 Hashing Is Mandatory

All PII fields (em, ph, fn, ln, ct, st, zp, country) must be SHA-256 hashed server-side before sending to Meta. The NowFit dev team implements this. Raw unhashed data must never be transmitted. Meta will reject unhashed PII and may penalize the dataset.

fbc / fbp: The Most Overlooked Parameters

fbc is derived from the fbclid URL parameter present when a user clicks a Meta ad. fbp comes from the _fbp browser cookie set by the Meta Pixel. Without these two values in the Stripe CAPI event, Meta cannot reliably attribute the purchase back to the specific ad that drove it. These values must be captured at the moment the lead submits the form on Monet's site and preserved through the NowFit CRM all the way to Stripe customer metadata.
