Tracking Scope & Operating Model

NOWFit should treat Monet's ecosystem as a multi-surface lead funnel. Both the WordPress site and the Netlify landing page function as lead and application intake first — with consultation, qualification, and payment happening downstream. Tracking should not stop at a raw lead form submission.

Critical PrincipleDo not optimize Meta campaigns for raw form fills if the business closes value later through consultations, qualification, and payment. Meta should receive the full signal hierarchy to optimize correctly.

Install the Browser Meta Pixel

Install the browser pixel on every active funnel surface. Use one method per surface only — do not combine multiple installation approaches on the same page.

🚫
One Pixel Per SurfaceNever install the browser pixel twice on the same page through different methods (e.g., manual head code AND the Meta plugin simultaneously). Duplicate installs cause duplicate PageView events and noisy Diagnostics.

WordPress Installation Options

A
Header Injection Plugin (Recommended for Non-Developers)
Install "Head & Footer Code" from WordPress plugins. Paste pixel code into the HEAD field.
Plugins › Add New › Head & Footer Code › Tools › HEAD field
B
Child Theme / wp_head Hook
Add pixel code before </head> in child theme's header.php, or via wp_head hook in functions.php.
C
Official Meta Plugin
Install the official Meta for WordPress plugin. Run the Facebook Business Extension setup flow. Ensure you select the existing data set, not create a new one.

Netlify / GHL Installation Options

A
GHL Funnel Head Tracking Code
Navigate to the funnel settings inside GoHighLevel. Paste the pixel base code into the Head Tracking Code field.
Sites › Funnels › Select Funnel › Settings › Head Tracking Code

Netlify Snippet Injection

The Conversions API requires an access token generated inside Events Manager and stored securely server-side. This token must never appear in front-end code or public repositories.

1
Open Events Manager
Navigate to Events Manager inside Meta Business Suite.
2
Select NOWFit Master Dataset
Confirm you are inside the correct data set (ID: 3015982851929798).
3
Open Settings → Conversions API
Go to the Settings tab and scroll to the Conversions API section.
Events Manager › Settings › Conversions API
4
Click Generate Access Token
Click Generate Access Token. Copy the full token string immediately — it will not be shown again.
5
Store Token Securely
Store in WordPress wp-config.php, Netlify environment variables, or a secure secret manager. Never in front-end HTML or JavaScript.
🔒
Token Security RuleThe CAPI access token must be stored server-side only. Never place it in browser-visible JavaScript, public HTML, or any Git repository. Exposure of this token compromises your entire Meta data set.

Event Deduplication

Deduplication is the most critical technical rule in this entire setup. Meta merges browser and server events into one conversion only when both share the same event_name and the same event_id. Without this, Meta counts every conversion twice.

Deduplication Flow
User Submits Form
↓
Browser fires
fbq('track', 'Lead', {}, {eventID: 'lead_12345'})
↓
Server sends
event_name: Lead / event_id: lead_12345
↓
Meta matches shared event_id → Deduplicates
↓
Single Conversion Counted ✓
Browser Event — Lead with event_id
// Generate unique ID for this lead event
const eventId = `lead_${Date.now()}_${Math.random().toString(16).slice(2)}`;

// Browser pixel fires with the event ID
fbq('track', 'Lead', { content_name: 'Main Application' }, { eventID: eventId });
Server Event (CAPI Payload) — same event_id
{
  "event_name": "Lead",
  "event_id":   "lead_12345",          // ← must match browser eventID exactly
  "event_time": 1713999600,
  "action_source": "website",
  "event_source_url": "https://bejewelled-llama-eaffb5.netlify.app/"
}

Recommended Event ID Convention

EventRecommended ID Format
Leadlead_<submission_id>
Scheduleschedule_<booking_id>
QualifiedLeadqualified_<stage_change_id>
Purchasepurchase_<order_id>
❌
Common Deduplication Mistakes • Browser fires Lead but server sends lead (case mismatch)
• Browser and server use different ID values
• Multiple pixel installs creating duplicate browser events
• Purchase fires before payment is complete

Advanced Matching Parameters

Advanced matching improves Meta's ability to connect events to real users. Send as many identifying parameters as your form captures. This directly improves attribution accuracy, audience modeling quality, and optimization performance.

🔐 Hash Before Sending (SHA-256)
email
phone
first_name
last_name
city
state
zip
country
✅ Send Raw — Do NOT Hash
client_ip_address
client_user_agent
fbc
fbp
⚠️
Hashing RuleHashing client_ip_address, client_user_agent, fbc, or fbp will break match quality. These must always be sent as raw unmodified values.
Advanced Matching — Full User Data Object
"user_data": {
  "em":                  sha256(trim(lowercase(email))),
  "ph":                  sha256(digits_only(phone)),
  "fn":                  sha256(trim(lowercase(first_name))),
  "ln":                  sha256(trim(lowercase(last_name))),
  "client_ip_address":   req.ip,           // raw — do not hash
  "client_user_agent":   req.headers['user-agent'],  // raw
  "fbc":                 cookie._fbc,      // raw
  "fbp":                 cookie._fbp       // raw
}

Revenue Signal Quality

Meta's optimization is directly determined by the quality of the conversion signals it receives. If only a Lead event is sent, Meta will optimize for cheap form submissions regardless of whether those leads become customers.

Funnel StageMeta EventWhy It Matters
Final application submittedLeadSignals intent, enables audience growth and lookalikes
Consultation or call bookedScheduleStronger intent signal, filters out passive submissions
Lead marked qualifiedQualifiedLeadTargets Meta optimization toward higher-value segments
Payment or enrollment completePurchaseRevenue signal — enables ROAS optimization and value-based lookalikes
🎯
Optimization StrategyOnce sufficient Purchase data exists, campaign optimization should move to Purchase as the primary optimization event. Until then, QualifiedLead is the best interim optimization target if the business can reliably define it.

6

Trigger a QualifiedLead stage update and a test purchase (where applicable). Confirm both events appear.

Diagnostics & Monitoring

After launch, monitor Events Manager → Diagnostics and individual event detail pages regularly. The five most common issues and their fixes are below.

IssueWhat It MeansFix
Duplicate eventsPixel installed more than once, or workflow trigger fires twiceRemove the extra browser pixel install or duplicate workflow action
Missing parametersServer payload is incomplete — required fields are absentMap required fields correctly in the CAPI payload before resending
Low event match qualityToo few user identifiers are being sent with each eventAdd email, phone, name, IP address, user agent, fbc, and fbp where available
Deduplication warningBrowser and server event IDs do not matchReuse the exact same event_id on both the browser pixel call and the CAPI payload
False purchase signalPurchase event is firing before payment is finalizedMove the Purchase trigger to a payment confirmation hook or webhook only
📊
Where to MonitorEvents Manager → select the data set → Diagnostics tab. Also review individual event detail pages for Event Match Quality scores and browser/server split breakdowns.
